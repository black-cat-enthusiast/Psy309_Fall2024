Magnitude of Manipulations: Assignment Instructions

1. Experiments: Manipulating Independent Variable(s)

If we want to go further from just showing an association to establishing that one construct directly influences or causes changes in another, we need to do experiments. In an experiments, we manipulate one variable to see if those unpredictable/independent manipulations create predictable effects on the dependent variable.

Think of a way in which you could manipulate an independent variable to create changes in your dependent variable of interest. In a short paragraph (3-5 sentences):

  • Discuss your idea for how you could manipulate one thing (the independent variable) to change the dependent variable.

  • Be clear on the direction of manipulation, e.g., if one increases how much an assignment is worth, a student’s nervous level will also increase

  • Provide an in-text citation to (at least) one peer-reviewed article that shows that the constructs might be related. If you can’t find evidence of the exact same manipulation, show an association and describe how you could change it into a manipulation, or show how the manipulation has been used in another context but could be applied here.

  • If there is no direct evidence or reporting of this relationship, then cite an article that shows that these constructs could be related.

  • Below the paragraph, provide an APA style reference to the paper that you cited.

2. Proposed Magnitude of the effect of your IV on the DV

When designing an experiment, the predicted magnitude of an effect is a key factor, as this can be used to determine the most appropriate amount of participants (the larger the proposed effect, the fewer participants would be needed to detect it).

Based on your literature searches, how big of an effect are you predicting the experimental manipulation that you outlined above would produce on the DV of interest?

  • Using numeric values and descriptive language, explain the predicted magnitude of your proposed IV on the DV of interest.

  • A good place to start would be to review findings using your DV in the literature. How varied are responses using your chosen measure? Do participants usually respond / perform in consistent ways? Or is there a lot of variability in responses? Do different studies that report this measure have similar mean values in their control groups? Understanding how much spread of scores you might expect to see in your control condition could help you think critically about what you would expect data from your experimental group to look like.

  • Remember, that the magnitude of the effect does not determine the importance of your findings - many important findings from the field of psychology involve quantification of small effects.

  • Use at least one peer-reviewed reference to support the proposed magnitude of the relationship between your IV and DV.

A Quick intro to R-Code and Object-oriented programming

Rstudio can operate like a high-level calculator. If we run a functional process in-line, the result of that process will be printed out:

67 * 88
[1] 5896

We could also “assign” the results of a functional process onto an object in the working environment (this is the basis of object-oriented programming).

# Object-oriented programming 
a <- 67 * 88

Here, we told Rstudio to assign the result of the function onto the object “a”. We could then perform subsequent calculations with the object “a”:

a + 200
[1] 6096

But much of the time, as analysts, we are working with more data than a single value. Rstudio can also assign complex data structures onto objects, which are used in subsequent computations.

b <- c(1,2,3,4,5) # Assign a list onto the object "b"

We could also assign a two-dimensional piece of data (i.e., with multiple rows and multiple columns) on to an object:

c <- data.frame(
  ID = c(1,2,3,4,5), 
  Score = c(100,110,90,95,107)
)

Then we could “pass” the object c to subsequent functions. For example, the head function shows us the first six rows of data:

head(c)
  ID Score
1  1   100
2  2   110
3  3    90
4  4    95
5  5   107

R is the statistician’s tool so many statistical functions are built in:

mean(c$Score)
[1] 100.4
  • Prints out the mean of the specified column from a dataframe.
sd(c$Score)
[1] 8.264381
  • Prints out the standard deviation of the specified column from a dataframe.

Random number generation

In this course, we will be generating fictitious data to represent your chosen project. In order to do this, we need to understand how random number generation works in R.

We should include a line of code set.seed() before computing any kind of pseudo-random process. Under the hood, Rstuido is not really using a random process. Instead, it is using a process that is finely-tuned to appear random. The set.seed() function specifies a starting point for the random number generation, which will ensure that the subsequent functions always produce the same values.

set.seed(123) # Beginning of the random process

Here, I will assign onto the object d a dataframe with two columns and 100. rows. The code below instructs Rstudio to fill the first column with the values 1 through 100. Method #2: Sample from a List of Values

The method above (sampling from a normal distribution) will generate data with partial values. In many cases, students are proposing measures that produce discrete data for their Psy309 projects. If you want to generate discrete data, use the code bellow to sample from a specified list of values.

This code assigns a dataframe with two columns and 100 rows onto the object e. The first column will contain the values 1 through 100. The second column will contain the values 1 through 7 randomly sampled with replacement.

e <- data.frame(
  ID = c(1:100),
  Score = sample(x = c(1:7), size = 100, replace = TRUE)
)

We could also control the likelihood of each outcome in the list using the code below. Here, I am first generating a separate object names prob_info which contains a list of values that is equal in length of the options that will be sampled (in this case, 7), and that all add up to 1. Then, I could pass the list of probabilities to the prob argument of the sample function:

Prob_info <- c(0.02, 0.03, 0.05, 0.1, 0.1, 0.4, 0.4)

f <- data.frame(
  ID = c(1:100),
  Score = sample(x = c(1:7), size = 100, replace = TRUE, prob = Prob_info)
)

We could use the same code as above (only changing the object that we will start with) to see a histogram of the data that we saved on the object f in the chunk above:

f %>%

  theme_classic()+
  scale_x_continuous(breaks=c(1:7))

Aggregate table of data

For Practical assignment #3, we want to see that you’ve created data that make sense for your chosen DV. To do this, we need to see a summary table of your data in addition to a preview of the raw values. The code below demonstrates how to create a good-looking summary table to showcase your simulated data.

table <- JLB_data %>% # Take the data AND THEN
  melt(id.vars = c("ID","Condition")) %>% # Switch to long form AND THEN
  group_by(Condition,variable) %>% # Group by the IVs
  summarise( # Calculate descriptive stats
    n = n(),
    mean = mean(value),
    sd = sd(value),
    min = min(value),
    max = max(value)
  ) %>% mutate(se = sd / sqrt(n)) # Create a column for SE, and compute it

# Print out the object table that the code above created
